Python进程和线程学习

发表于 2019-08-08

CPU+RAM+各种东西（显卡，硬盘，键盘鼠标）构成计算机，但计算机的运行基本上和CPU，寄存器，RAM挂钩。

CPU的处理速度相当快，对于单核CPU而言，多任务处理，基本上是让系统轮流交替执行各任务，比如有A，B，C，D四个任务，怎么让他们同时执行呢？

CPU处理也是一步一步来的，任务A执行0.01s，任务B执行0.01s，任务C执行0.01s，最后任务D执行0.01s…如此循环往复，由于CPU的速度太快了，看上去就像是同时执行一样。

进程和线程

对于操作系统而言，一个任务就是一个进程（Process），打开QQ，开了一个进程，打开LOL，又开了一个进程。

大部分进程不止干一件事情，比如QQ，可以打字，同时还可以视频，所以一个进程要运行多个子任务，这每个子任务就称为线程（Thread）。

多个任务的实现有3种方式

  • 多进程，启动多个进程，每个进程只有一个线程
  • 多线程，启动一个进程，但在一个进程内启动多个线程
  • 多进程+多线程，启动多个进程，每个进程再启动多个线程

Python既支持多进程，又支持多线程。

Unix/Linux下实现多进程

Unix/Linux操作系统提供了fork()系统调用，普通函数调用一次，返回一次，fork()调用一次，返回两次，子进程永远返回‘0’，父进程返回子进程的ID，这样，一个父进程可以fork出很多子进程，父进程需要记住每个子进程的ID，子进程调用getppid()就可以得到父进程的ID
